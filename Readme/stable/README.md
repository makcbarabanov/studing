# Readme/stable — снимки рабочей разметки

**Назначение:** хранить **копии** одобренных кусков UI (HTML + минимальные стили) как **якорь** — быстрее доставать «как было, когда работало», чем копаться в истории Git.

## Правила

1. **Когда класть сюда:** по явному одобрению (например, после итерации «таблица мечт — ок») или по запросу.
2. **Имя файла:** говорящее, одна зона = один файл, например `dreams_table.html`, `header_step_row.html` (по мере появления).
3. **Содержимое:** самодостаточный фрагмент: `DOCTYPE` не обязателен, но **структура + классы + ключевой CSS** — да, чтобы вставить/сверить в `index.html` или в тестовую страницу.
4. **Источник правды** по смыслу и цифрам — **[Readme/UI-standards.md](../UI-standards.md)** (для таблицы мечт — **§ 1.1**). Файлы в `stable/` — **снимок**, не замена стандарта.
5. **Дублирование с Git:** снимок не отменяет теги/коммиты; это удобный слой для владельца репо без обязательного GitHub-скилла.

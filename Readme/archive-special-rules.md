# Архив спец-правил (финцели и книги)

Дата архивации: 2026-04-10.

## Зачем архив

Спец-правила для финансовых целей и книг дали слишком много побочных эффектов в интерфейсе и поддержке. Текущий рабочий режим проекта: единый общий вид мечт и шагов без специальных таблиц и формул.

## Что было в спец-логике

- Финцели:
  - отдельная модалка создания шагов по месяцам;
  - таблица в аккордеоне с колонками "Месяц | ВЗНОС (План/Факт) | СУММА (План/Факт) | Итог";
  - вычисления по `plan_amount` и `fact_amount`.
- Книги:
  - отдельный модуль `books_reading`;
  - дополнительные элементы в `GET /schedule` на основе `dream_books` и `dream_books_log`.

## Текущее состояние (после упрощения)

- UI использует единый рендер обычных шагов.
- Спец-таблица финцели не применяется в боевом режиме.
- Книжные элементы не добавляются в `GET /schedule` (флаг в `main.py`).

## Где лежит старый код

- Frontend: `index.html`
  - флаг `ENABLE_SPECIAL_STEPS_BOOKS_FINANCE`.
  - сохранена логика модалки и таблицы финцели.
- Backend: `main.py`
  - флаг `ENABLE_SPECIAL_BOOKS_IN_SCHEDULE`.
  - сохранены эндпоинты книг и `POST /dreams/{dream_id}/steps/finance`.

При необходимости можно вернуть спец-режим через флаги и отдельное тестирование.

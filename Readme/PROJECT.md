## Миссия

Социальная сеть для целеустремлённых людей. Платформа, где мечты превращаются в реальность через поддержку окружения и декомпозицию целей.

## Функционал (MVP)

### Витрина мечт
Пользователь описывает желания в ЛК; мечты могут стать публичными; другие пользователи видят витрину и могут помогать.

### Путь к цели
Большая мечта разбивается на шаги и расписание. AI-наставник (бот) — отчёты, прогресс, коррекция траектории.

- **Папка:** `db_backups/` в корне проекта.

## Правила взаимодействия

1. Обращение — на «ты».
2. **Единственный источник правды — БД.** Без заглушек; изменения UI с изменением данных — только через API и БД.
3. **Пропорции вёрстки:** контенту — максимум места; хедер/меню — компактно.
4. **Strict Scope по умолчанию:** только явно запрошенный объём; сопутствующие правки — после согласования.

## Правила разработки

1. **Безопасность:** `.env` не в Git.
2. **Git:** локальные проверки → commit → push.
3. **Стиль:** читаемый код, обработка ошибок.
4. **Документация БД:** изменения схемы — сразу в [tables.md](tables.md).
5. **История изменений:** значимые изменения — пункт в [CHANGELOG.md](CHANGELOG.md); схема БД — tables.md; roadmap — в БД / `roadmap.html`. Правило: [.cursor/rules/update-readme.mdc](../.cursor/rules/update-readme.mdc).
6. **Телефоны:** формат без пробелов, `+7999...`; скрипт `scripts/normalize_phones_and_passwords.py`.
7. **Миграции `_sql/`:** файлы в Git до выката; Forge по возможности сам запускает `run_migrate.py` (из контейнера `app` или с хоста — см. RUNBOOK). Устаревший бэкенд на сервере лечится **деплоем** актуального кода и пересборкой контейнера по RUNBOOK (не ручным правкой сервисов).
8. **Стратегия важнее тактики:** системные проблемы — остановка и предложение системного решения.
9. **Папка `errors/`:** после фикса — предложить удалить артефакт по подтверждению.
10. **Деплой:** только `prod-safe` код через Git; данные БД не через pull.
11. **Уточнения:** при неясной задаче — сначала вопросы.
12. **Модалка шага** (`#modal-add-step`): «Повторять» vs серия — канон в этом репозитории и [UI-standards.md](UI-standards.md); см. также CHANGELOG (п. 12 и связанные).
13. **`_trash/`:** устаревшие файлы — в локальную корзину `_trash/` (в `.gitignore`); не раз в месяц чистить без политики Макса. Важное сохранять коммитами до переноса в корзину.
14. **Прод-миграции:** additive; разрушительные операции — только по явному согласованию.

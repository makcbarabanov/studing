# UI-стандарты Острова

**Назначение:** единый источник правды по повторяющимся визуальным паттернам в ЛК и связанных страницах. При новом меню, переключателе или **поле даты** — сначала сверка с этим файлом, затем переиспользование тех же классов/токенов, а не копирование «на глаз».

**Где в коде:** основной фронт кабинета — `index.html` (встроенные `<style>` и скрипты).

---

## 1. Сегменты-фильтры (активный = зелёный фон)

**Эталон:** блок фильтра мечт / шагов — кнопки `.dreams-filter-btn`, активная `.dreams-filter-btn.active`.

| Состояние | Фон | Текст | Скругление |
|-----------|-----|-------|------------|
| Неактивный | `transparent` (по смыслу), лёгкая прозрачность | `#555` | `4px` |
| Hover | — | темнее | — |
| **Активный** | **`#4CAF50`** | **`#fff`**, `font-weight: 600` | `4px` |

Классы и отступы см. в `index.html` в секции `.dreams-filter-btn` / `.dreams-filter-btn.active`.

**Применение:** любые горизонтальные «вкладки-чипы» в одном стиле с фильтром мечт (не подчёркивание как у ссылки, а заливка активного пункта).

---

## 2. Навигация кабинета «мечты | шаги | витрина»

**Классы:** `.cabinet-nav-link` на элементах `#header-link-dreams`, `#header-link-steps`, `#header-link-showcase`; активный пункт — `.cabinet-nav-link.active`.

**Логика активного пункта** (функция `updateCabinetNavActive` в `index.html`):

- **мечты** — активен при разделах `dreams` и `buddy`;
- **шаги** — при `steps` и `steps-buddy`;
- **витрина** — при `showcase`;
- при `notifications`, `profile-edit` — ни один из трёх не подсвечивается (это не основные вкладки кабинета).

## 3. Календарь даты (стандарт «Остров»)

**Эталон в коде:** один общий попап `#island-date-calendar-popover` и функции `uiDateToIso` / `isoToUiDate` / `initIslandDateCalendar` в `index.html`.

### Ввод и формат

| Правило | Значение |
|--------|----------|
| Поле ввода | `type="text"`, класс **`island-date-field`**, плейсхолдер **`дд.мм.гг`**, `maxlength="10"`, `inputmode="numeric"`, `autocomplete="off"` |
| Отображение и ввод | **дд.мм.гг** (день и месяц с ведущим нулём при нормализации; год — **2 или 4 цифры**, правила года см. `uiDateToIso`) |
| В API и БД | только **ISO** `YYYY-MM-DD`; перед отправкой — **`uiDateToIso`**, для показа из API — **`isoToUiDate`** |
| Ошибка ввода | класс **`island-date-field-invalid`** на поле (красная рамка) |

**Не использовать** нативный `type="date"` для полей ЛК в этом стандарте: формат и локаль зависят от ОС/браузера и не совпадают с дд.мм.гг.

### Поведение календаря

- **Открытие:** при **фокусе** на поле с `island-date-field` (отдельная кнопка «календарь» не нужна).
- **Позиция:** `position: fixed` под полем или над ним, если снизу не хватает места (`islandCalPosition`).
- **Закрытие:** `Escape` (сначала календарь, затем модалка — см. обработчик `keydown` в `index.html`), клик вне попапа и вне активного поля, **mousedown** с учётом ссылок-подсказок `.modal-deadline-hints` (клик по подсказке не закрывает календарь), `blur` поля с небольшой задержкой, если фокус не ушёл внутрь попапа.
- **Сетка:** неделя с **понедельника**; шапка — кнопки «‹ ›», выпадающие **месяц** (`#island-cal-month`) и **год** (`#island-cal-year`), сетка дней `#island-cal-grid`, классы ячеек **`island-cal-day`**, пустые — **`island-cal-day-empty`**, выбранный день — **`island-cal-day-selected`**.
- **Месяцы в селекте:** краткие подписи из массива **`ISLAND_CAL_MONTH_SHORT`** (Янв–Дек), полное название в атрибуте **`title`** у option.
- **Годы:** диапазон строится вокруг текущего просмотра и «сегодня» (см. `islandCalFillYearSelect`), глобальный коридор **1980–2120**.
- При вводе с клавиатуры, если дата распознана, месяц/год в попапе **синхронизируются** (`islandCalRefreshFromTyping`).

### Внешний вид (палитра)

- Контейнер попапа: класс **`island-date-calendar-popover`** — светлый фон с лёгким голубым градиентом, рамка **`#81d4fa`**, тень с синим оттенком (как в `index.html`).
- Селекты месяца/года: **`island-cal-month-select`** / **`island-cal-year-select`** — белый фон, рамка **`#64b5f6`**, текст **`#01579b`**, жирный шрифт.
- Активные элементы и акценты: тёмно-синие **`#01579b`**, выделение дня **`#002171`** (см. стили `.island-cal-day` в `index.html`).

### Где уже применяется

- Дедлайн мечты (создание / редактирование): поля `#new-dream-deadline`, `#edit-dream-deadline`.
- Модалка шага: `#modal-add-step-date-single`, `#modal-add-step-date-from`, `#modal-add-step-date-to`.
- Модалка финансовых шагов: `#modal-fin-end-date`.

Любое новое поле даты в кабинете — **тот же маркер `island-date-field`** и тот же попап (после вызова `initIslandDateCalendar` поля, уже в DOM при загрузке, привязываются автоматически; для полей, добавляемых динамически позже, нужно либо вызывать ту же логику привязки, либо держать поля в статическом HTML до `initIslandDateCalendar`).

## 4. Как добавлять новые стандарты

1. Описать паттерн здесь (таблица или короткий список свойств).
2. Вынести общие правила в **один** набор классов в `index.html` (или позже — в общий CSS, если вынесем стили).
3. В `Readme/Readme.md` в блоке «Что сделано» — одна строка со ссылкой на этот файл (и при необходимости кратко — что именно добавлено в стандарты).

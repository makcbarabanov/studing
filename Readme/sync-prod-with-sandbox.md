# Безопасная синхронизация песка и прода

**Цель:** переносить на прод только безопасные изменения кода, не затрагивая данные БД и не ломая работающий сервер.

---

## Главный принцип

- **Git = только код и документация.**
- **БД песка и БД прода синхронизируются только дампами вручную, не через `git pull`.**
- **Продагент деплоит только проверенные коммиты `prod-safe`.**

---

## Шаг 1. Песок (Forge): подготовка безопасного релиза

В песке (локально) перед пушем:

```bash
cd /home/makc/Apps/island
git status
git diff --stat
```

Дальше:

1. Отдели экспериментальные изменения от релизных.
2. В релиз пускай только файлы, безопасные для прода.
3. Коммит помечай префиксом `prod:` (или `sandbox:` для песочных экспериментов).
4. Пуш в `main` только после проверки списка файлов.

---

## Шаг 2. Прод (Продагент): безопасный деплой

На проде перед обновлением всегда:

```bash
cd /home/makc/Apps/island
git status
git rev-parse --short HEAD
git fetch origin
git log --oneline HEAD..origin/main
```

Если `git status` не чистый:
- сделать `git stash push -u -m "pre-deploy-<date>"`,
- только потом обновляться.

Обновление:

```bash
git pull --ff-only origin main
```

Перезапуск приложения (по реальному прод-регламенту):

```bash
sudo systemctl restart island
# или нужный service name на проде
```

---

## Шаг 3. Smoke-check после деплоя

```bash
curl -sk https://islanddream.ru/index.html | sed -n '1038,1046p'   # версия
```

Проверить руками в браузере:
- логин,
- загрузка мечт,
- шаги,
- аватар,
- ключевой сценарий дня.

---

## Шаг 4. Откат (если что-то пошло не так)

```bash
cd /home/makc/Apps/island
git reflog -10
git reset --hard <предыдущий_коммит>
sudo systemctl restart island
```

---

## Роль Продагента (чеклист)

Продагент делает только:

1. Проверка diff между текущим `HEAD` и `origin/main`.
2. Подтверждение, что релиз `prod-safe`.
3. `git pull --ff-only` и restart сервиса.
4. Smoke-check и отчёт Максу.

Продагент не делает:
- ручных правок кода на проде,
- смешивания песочных и продовых конфигов,
- автоматического переноса БД песка в прод.

---

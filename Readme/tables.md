# Таблицы базы данных (ОСТРОВ)

Описание таблиц, используемых приложением. Структура восстановлена по миграциям (`_sql/`), `init.sql` и коду `main.py`. В реальной БД могут быть дополнительные колонки (например, в `dreams` — из импортов или других систем).

---

## Действующие таблицы

### 1. `users`

**Назначение:** Пользователи платформы. Учётные данные для входа (телефон, хеш пароля), профиль (имя, фамилия, город). Используется при логине, регистрации и в админке.

| Колонка         | Тип           | Описание |
|-----------------|---------------|----------|
| `id`            | SERIAL PRIMARY KEY | Уникальный идентификатор пользователя. |
| `name`          | VARCHAR(100)  | Имя. |
| `surname`       | VARCHAR(100)  | Фамилия. В API часто склеивается с именем в `full_name`. |
| `phone`         | VARCHAR(20) UNIQUE | Телефон (уникальный), используется как логин. |
| `city`          | VARCHAR(100)  | Город. |
| `password_hash` | TEXT          | Хеш пароля (bcrypt) или устаревшее значение в открытом виде. |
| `avatar_path`   | VARCHAR(255) NULL | Путь к файлу аватарки относительно каталога `media/` (например `avatars/123.jpg`). Файлы лежат в `media/avatars/`. Миграция mig_012. |
| `buddy_id`      | INT NULL           | ID бадди (другой пользователь). Связь один-на-один: бадди может просматривать мечты этого пользователя в разделе «Бадди». Взаимно: у бадди в `buddy_id` указывается этот пользователь. Миграция mig_013. |
| `buddy_trust`   | BOOLEAN DEFAULT false | Уровень доверия к бадди: true — бадди может редактировать мечты этого пользователя (статус, шаги, удаление и т.д.); false — только просмотр. Миграция mig_014. |

### 2. `dreams`

**Назначение:** Мечты участников. Основная сущность личного кабинета: текст мечты, владелец (`user_id`), дедлайн, цена, статус и категория (связь со справочниками), шаги подгружаются из `dream_steps`. Таблица используется в GET/POST/PATCH/DELETE мечт.

| Колонка           | Тип            | Описание |
|-------------------|----------------|----------|
| `id`              | SERIAL PRIMARY KEY | Уникальный идентификатор мечты. |
| `date`            | DATE / TIMESTAMP   | Дата создания мечты. В миграциях: `date TIMESTAMP DEFAULT NOW()` или `DATE DEFAULT CURRENT_DATE`. NOT NULL. |
| `user_id`         | INT NOT NULL   | Владелец мечты, ссылка на `users.id`. |
| `dream`           | TEXT NOT NULL  | Текст мечты. Основное поле, отображается в таблице и в модалках. |
| `what_need`       | JSONB          | Что нужно для осуществления (массив). Может присутствовать в БД из других систем, приложением ОСТРОВ не используется. |
| `interference`    | JSONB          | Что мешает осуществить (массив). Не используется приложением. |
| `completion_count`| INT            | Количество раз, когда мечта была воплощена. Не используется приложением. |
| `helpers_id`      | JSONB          | Массив ID пользователей, которые помогли. Не используется приложением. |
| `created_at`      | TIMESTAMP      | Дата/время создания записи в БД. Может быть в БД, в миграциях проекта нет. |
| `updated_at`      | TIMESTAMP      | Дата/время последнего обновления. Может быть в БД, в миграциях проекта нет. |
| `price`           | NUMERIC(12,2) / VARCHAR(50) | Цена (руб.). В миграции — NUMERIC; в некоторых БД — VARCHAR. В API отдаётся числом. |
| `title`           | VARCHAR(500)  | Краткий заголовок. Добавлен в mig_001; в API текст мечты берётся из `dream`, не из `title`. |
| `description`     | TEXT          | Расширенное описание. В API не используется. |
| `image_url`       | TEXT          | Ссылка на изображение. В API не используется. |
| `is_public`       | BOOLEAN DEFAULT true | Приватность: true = мечта открытая (попадает в витрину), false = приватная (анонимная, не в витрине). По умолчанию все открытые. Миграция mig_011. |
| `progress`        | INT DEFAULT 0 | Прогресс (0–100 или иная шкала). В API не используется. |
| `deadline`        | DATE          | Дедлайн (дата, до которой нужно осуществить). В формате YYYY-MM-DD, в API отдаётся строкой. |
| `status_id`       | INT NOT NULL DEFAULT 1 | Ссылка на `dreams_statuses.id`. Добавлен в mig_006. |
| `category_id`     | INT NULL      | Ссылка на `dreams_categories.id`. Добавлен в mig_006. NULL — без категории. |

**Примечание:** Колонки `status` и `category` (VARCHAR) после mig_006/mig_006b удалены; вместо них используются `status_id` и `category_id`. Колонка `is_public` гарантируется миграцией mig_011 (ADD COLUMN IF NOT EXISTS, по умолчанию true).

### 3. `dreams_log`

**Назначение:** Лог исполнений мечт — кто и когда отметил, что мечта сбылась. Одна мечта может сбываться многократно (разные люди или повторно). Используется для статистики в хедере: «Сбылось: N мечта, M раз». Миграция mig_016.

| Колонка                | Тип            | Описание |
|------------------------|----------------|----------|
| `id`                   | SERIAL PRIMARY KEY | Уникальный идентификатор записи. |
| `dream_id`             | INT NOT NULL   | Мечта. REFERENCES `dreams(id)` ON DELETE CASCADE. |
| `date`                 | DATE NOT NULL  | Дата исполнения. |
| `fulfilled_by_user_id` | INT NOT NULL   | Кто отметил исполнение. REFERENCES `users(id)`. |

Индексы: `idx_dreams_log_dream_id`, `idx_dreams_log_fulfilled_by`.

---

### 4. `dreams_steps`

**Назначение:** Шаги к мечте (чек-лист). Каждая запись привязана к одной мечте (`dream_id`), имеет название, флаг «выполнено» и порядок сортировки. При удалении мечты шаги удаляются каскадно (ON DELETE CASCADE).

| Колонка    | Тип            | Описание |
|------------|----------------|----------|
| `id`       | SERIAL PRIMARY KEY | Уникальный идентификатор шага. |
| `dream_id` | INT NOT NULL   | Мечта, к которой относится шаг. REFERENCES `dreams(id)` ON DELETE CASCADE. |
| `title`    | VARCHAR(500) NOT NULL | Название шага. |
| `completed`| BOOLEAN DEFAULT false | Выполнен ли шаг (галочка в интерфейсе). |
| `sort_order` | INT DEFAULT 0 | Порядок отображения (меньше — выше в списке). |
| `deadline`   | DATE NULL     | Дедлайн шага (одна дата «сделать до»). Миграция mig_015. |
| `deleted`    | BOOLEAN DEFAULT false | Мягкое удаление: true — шаг скрыт из текущих, показывается в разделе «Шаги» в блоке «Удалённые» с возможностью восстановить. Миграция mig_015. |
| `plan_amount` | NUMERIC(12,2) NULL | Для шагов финцели: плановая сумма за период (например, 17 000 ₽ в месяц). Округление до «красивых» сумм задаётся в `steps_rules` (например, `plan_round: thousands`). |
| `fact_amount` | NUMERIC(12,2) NULL | Для шагов финцели: фактически внесённая сумма за период. Редактируется пользователем; колонка «Итог» (профицит/дефицит/по плану) считается по плану и факту. |

Индекс (не таблица): `idx_dreams_steps_dream_id` на столбец `dreams_steps(dream_id)` — для быстрого поиска шагов по мечте (mig_001, имя обновлено в mig_017). Всё, что касается мечт, имеет префикс `dreams_`.

### 5. `dreams_statuses`

**Назначение:** Справочник статусов мечты. Используется для отображения и смены статуса в таблице и в модалках. Значения по умолчанию заданы в mig_005.

| Колонка   | Тип               | Описание |
|-----------|-------------------|----------|
| `id`      | SERIAL PRIMARY KEY | Уникальный идентификатор. В `dreams.status_id` хранится это значение. |
| `code`    | VARCHAR(50) UNIQUE NOT NULL | Код статуса: `planned`, `in_progress`, `done`. |
| `label_ru`| VARCHAR(100) NOT NULL | Подпись на русском: Запланировано, В работе, Выполнено. |
| `icon`    | VARCHAR(20)       | Иконка (эмодзи): 💡, ☭, ✅. |

Начальные данные (mig_005): planned — Запланировано 💡; in_progress — В работе ☭; done — Выполнено ✅.

**Статусы мечты (аналитика, отображение «мечта сбылась N раз»):**  
В колонке статусов мечты показывается не только иконка текущего статуса, но и число исполнений (если мечта сбывалась). Чтобы не нарушать ширину столбца: при 1–98 — иконка (галочка) + цифра; при 99–999 — **серебряный** статус (цифра «99+» / «100+» и т.д. в серебряном оформлении); при 10000 и выше — **золотой** статус мечты (золотая цифра «10000+»). Конкретные пороги и иконки уточняются в интерфейсе.

### 6. `dreams_categories`

**Назначение:** Справочник категорий мечты. Используется для выбора категории при создании/редактировании мечты и отображения в таблице. Значения по умолчанию заданы в mig_005.

| Колонка   | Тип               | Описание |
|-----------|-------------------|----------|
| `id`      | SERIAL PRIMARY KEY | Уникальный идентификатор. В `dreams.category_id` хранится это значение. |
| `code`    | VARCHAR(50) UNIQUE NOT NULL | Код категории: health, finance, relations, charity, projects, development. |
| `label_ru`| VARCHAR(100) NOT NULL | Подпись на русском. |
| `icon`    | VARCHAR(20)       | Иконка (эмодзи). |

Начальные данные (mig_005): health ❤️ Здоровье; finance 💰 Финансы; relations 💑 Отношения; charity 🤝 Благотворительность; projects 📋 Проекты; development 📚 Развитие.  
Дополнительно (mig_008): иконки для Семья 🏠 и Прочее 📦; добавлена категория sport ⚽ Спорт.  
(mig_009): добавлена категория spirituality 🙏 Духовность.  
(mig_010): добавлена категория achievements 🏆 Достижения.

### 7. `buddy_requests`

**Назначение:** Запросы «стать бадди». Пользователь A отправляет запрос пользователю B; B при заходе в кабинет видит запрос и может принять или отклонить. При принятии у обоих выставляется `users.buddy_id`. Миграция: `_sql/mig_buddy_requests.sql`.

| Колонка       | Тип            | Описание |
|---------------|----------------|----------|
| `id`          | SERIAL PRIMARY KEY | Уникальный идентификатор запроса. |
| `from_user_id`| INT NOT NULL   | Кто отправил. REFERENCES users(id) ON DELETE CASCADE. |
| `to_user_id`  | INT NOT NULL   | Кому отправлен запрос. REFERENCES users(id) ON DELETE CASCADE. |
| `status`      | VARCHAR(20) NOT NULL DEFAULT 'pending' | pending, accepted, declined, cancelled. |
| `created_at`  | TIMESTAMP WITH TIME ZONE DEFAULT NOW() | Дата создания. |

Индексы: `idx_buddy_requests_to_user`, `idx_buddy_requests_from_user`.

---

### 8. `steps_rules`

**Назначение:** Справочник правил разбиения целей на шаги. Позволяет описывать типовые схемы (например, финансовые цели с годовой суммой, равномерно разбитой по месяцам) и применять их при создании мечты.

| Колонка        | Тип               | Описание |
|----------------|-------------------|----------|
| `id`           | SERIAL PRIMARY KEY | Уникальный идентификатор правила. |
| `category_code`| VARCHAR(50) NOT NULL | Код категории из `dreams_categories.code` (например, `finance`). |
| `rule_code`    | VARCHAR(100) NOT NULL | Внутренний код правила (например, `yearly_amount_by_month_equal`). |
| `rules`        | JSONB NOT NULL    | JSON с параметрами правила (описание, период, количество шагов, способ распределения и т.п.). |
| `comment`      | TEXT              | Человекочитаемый комментарий.

Пример записи: правило `yearly_amount_by_month_equal` для категории `finance` задаёт схему «годовая сумма, равномерно разбитая на 12 ежемесячных шагов календарного года».

## Актуальные таблицы (без префикса _old_)

Приложение ОСТРОВ использует: **users**, **dreams**, **dreams_log**, **dreams_categories**, **dreams_statuses**, **dreams_steps**, **buddy_requests**, **steps_rules**. Остальные таблицы в схеме `public` считаются неиспользуемыми.

## Таблицы с префиксом _old_

Миграция **mig_018** переименовывает все таблицы в `public`, которые не входят в список актуальных выше, добавляя префикс **`_old_`** (например: `darts` → `_old_darts`). Так наглядно выделяются актуальные таблицы. Если старая таблица снова понадобится, префикс убирают вручную (в DBeaver или миграцией `ALTER TABLE _old_<name> RENAME TO <name>`). Таблицы, уже начинающиеся с `_old_`, миграция не трогает.

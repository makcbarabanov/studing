# Правила и инструкции

**Назначение документов в Readme и записей в `steps_rules`:** передавать контекст так, чтобы новый админ или агент (например, новый участник проекта) мог молча ознакомиться с инструкциями и выполнить задачу без искажений — как если бы это делали авторы проекта, уже находящиеся в контексте. Инструкция — не просто «решить задачу», а полноценная передача контекста.

- **Текстовые инструкции** (пошаговое описание: как создавать мечту и шаги для финансовой цели, для цели прочтения книг и т.д.) хранятся **здесь, в Readme/rules.md**. Каждая особый тип цели (rule_code) имеет свой раздел ниже.
- В таблице **`steps_rules`** в БД хранятся: **rule_code**, **category_code**, **rules** (JSONB с параметрами для кода) и **comment** (краткое описание или ссылка на раздел в rules.md). Код приложения по rule_code мечты находит правило и применяет логику; человек или агент для полного контекста читает соответствующий раздел в rules.md.
- Цепочка «мечта → шаги (простые и особые) → расписание» и то, как система по `dreams.rule_code` выбирает инструкцию при «Добавить шаг», описаны в [Readme/structure.md](structure.md) (раздел «Цепочка: мечта → шаги → расписание»).

---

## Мечты по умолчанию закрыты для витрины

**Все новые мечты** создаются с **is_public = false**: по умолчанию мечта **не попадает на витрину** (закрыта от публичного показа). Пользователь при желании сам открывает мечту для витрины через настройки в кабинете. Скрипты сидирования и API при создании мечты без явного `is_public` должны использовать **is_public = false**.

# Инструкция: создание шагов для финансовой цели

Документ описывает разбиение **финансовой цели на помесячные шаги** (например, «накопить 200 тыс ₽», «заработать 3 млн»). Нужную сумму за период можно **заработать или накопить**, распределив её **равными долями по месяцам** (как у Кости: по 17 т.р.) либо **разными суммами** (например: январь 0, февраль 100 т.р., далее каждый месяц = предыдущий × N, сумма с февраля по декабрь = 3 млн). Интерфейс один и тот же: таблица ВЗНОС/СУММА/Итог в мечте, в расписании — строка «Внеси N т.р.» на каждый месяц.

---

## 1. Правила в справочнике

- Таблица правил: **`steps_rules`** (см. [Readme/tables.md](tables.md)).
- **Равные доли по месяцам** (пример: Костя — 17 т.р. в месяц):
  - **category_code:** `finance`
  - **rule_code:** `yearly_amount_by_month_equal`
  - **rules** (JSONB): **plan_round** `"thousands"`, период по умолчанию — календарный год (12 месяцев), может быть задан пользователем.
- **Разные суммы по месяцам** (пример: январь 0, февраль 100 т.р., март = февраль × N, … сумма за период = 3 млн):
  - **category_code:** `finance`
  - **rule_code:** `yearly_amount_by_month_custom`
  - **rules** (JSONB): параметры для расчёта или хранения явного списка сумм по месяцам (см. п. 3 и раздел «Модалка при „Добавить шаги“» ниже).

При добавлении новых вариантов (например, своя формула роста) в `steps_rules` добавляется строка с новым `rule_code` и своими параметрами в **rules**.

---

## 2. Создание мечты (цели)

- Создать запись в **`dreams`**:
  - **user_id** — владелец цели.
  - **dream** — текст цели (например, «От 200 тыс ₽ на депозит к концу года»).
  - **category_id** — соответствует категории «Финансы» (в `dreams_categories` код `finance`, id можно посмотреть по справочнику).
  - **deadline** — дата конца периода (например, 31.12.2026).
  - **price** — **целевая сумма в рублях** (например, 200000). Она используется в интерфейсе для отображения прогресса в формате «X/200 т.р.» и для логики итога. Если не указать, будет использоваться сумма планов по шагам (12 × план месяца).
  - **status_id**, **is_public** — по умолчанию как у обычной мечты.

---

## 3. Расчёт плана по месяцам

- **Целевая сумма** берётся из **dream.price** (если задана и > 0) или считается по сумме планов по шагам.
- **Равные доли** (`yearly_amount_by_month_equal`): целевая сумма делится на число месяцев периода; при **plan_round: "thousands"** округление до тысяч (200 000 / 12 → **17 000** ₽ в месяц).
- **Разные суммы** (`yearly_amount_by_month_custom`): планы по месяцам задаются явным списком (поля по каждому месяцу в модалке) или формулой. Пример: первый месяц 0, второй месяц A, каждый следующий = предыдущий × N, сумма за период = целевая — тогда N подбирается так, чтобы сумма с 2-го по последний месяц равнялась цели (например, янв 0, фев 100 000, март–дек с множителем так, чтобы фев+…+дек = 3 000 000).

---

## 4. Создание шагов в `dreams_steps`

Для каждого месяца календарного года (январь — декабрь) создаётся одна запись в **`dreams_steps`**:

| Поле         | Значение |
|--------------|----------|
| **dream_id** | id созданной мечты |
| **title**    | Название месяца: «Январь», «Февраль», …, «Декабрь» |
| **plan_amount** | Месячный план в рублях (например, 17000) |
| **fact_amount** | 0 (пользователь будет заполнять) |
| **sort_order** | 1–12 (порядок месяцев) |
| **deadline** | Последний день месяца (например, 31.01.2026, 28.02.2026, …) |
| **completed** | false |
| **deleted**  | false |

Всего — по одной записи на каждый месяц периода. Для правила «разные суммы» в **dreams_steps.plan_amount** подставляются значения из анкеты (явный список или рассчитанные по формуле).

---

## 4a. Модалка при «Добавить шаги» (анкета финцели)

Когда у мечты категория «Финансы» и пользователь нажимает **«Добавить шаги»**, показывается модалка с вопросами. Ответы сохраняются в настройках мечты (**dreams.settings**) и/или в параметрах правила; по ним создаются шаги в **dreams_steps** и задаётся дата появления в расписании.

1. **Заработать / Накопить** — выбор формулировки (влияет на подписи в интерфейсе, например «Внести» vs «Заработать»).
2. **Нужная сумма** — целевая сумма в рублях (соответствует **dream.price**).
3. **Срок:**
   - Поле с **датой окончания** (дедлайн периода).
   - Подсказки-кнопки: **«12 месяцев»** — подставить в поле даты «сегодня + 12 месяцев»; **«До конца года»** — подставить 31.12 текущего года (или следующего, если текущий месяц близок к концу). По сроку определяется количество месяцев и список шагов (январь–декабрь или от текущего месяца до дедлайна).
4. **Доли: равные / разные:**
   - **Равные** — сумму делить на число месяцев; подсказка показывает результат (например: «≈ 250 000 ₽ в месяц»). Соответствует правилу `yearly_amount_by_month_equal`.
   - **Разные** — ниже (или на следующей карточке/экране) появляется блок с **полями по каждому месяцу** периода: количество полей = количеству месяцев в сроке; пользователь вводит плановую сумму за каждый месяц. Либо поддерживается режим «формула»: первый месяц 0, второй месяц A, множитель N — система считает остальные месяцы так, чтобы сумма за период равнялась нужной. Соответствует правилу `yearly_amount_by_month_custom`.
5. **Число внесения** — число каждого месяца (1–31), в которое в расписании появляется запись «Внеси N т.р.» (дедлайн шага = это число в соответствующем месяце). Например, 25 — шаг по марту показывается с дедлайном 25.03. По умолчанию можно использовать последний день месяца (31, с учётом 28/29 февраля).

После подтверждения модалки создаются записи в **dreams_steps** (title = название месяца, plan_amount из расчёта или из полей, deadline = выбранное число каждого месяца), мечте при необходимости проставляется **dreams.rule_code** и **dreams.settings** (срок, тип долей, число внесения, формула при «разные»).

---

## 5. Проверка в интерфейсе

- В личном кабинете владельца открыть мечту и раскрыть аккордеон шагов.
- Для финцели отображается таблица: **Месяц | ВЗНОС (План / Факт) | СУММА (План / Факт) | Итог**.
- **СУММА** показывается в формате «X/200 т.р.» (X — накопительный план или факт в тысячах, 200 — цель в тысячах из **dream.price**).
- **Итог:** для текущего месяца при отставании — «Внеси N т.р.»; для прошедших месяцев — «Нужно ещё N т.р.» (N — накопительный план к концу того месяца); для будущих месяцев — пусто.
- Редактируется только **ВЗНОС Факт** (фактический взнос за месяц); остальное считается автоматически.

---

## 6. Краткий чек-лист (ручное создание)

1. Убедиться, что в `steps_rules` есть нужное правило: `yearly_amount_by_month_equal` (равные) или `yearly_amount_by_month_custom` (разные).
2. Создать мечту в `dreams` (категория Финансы, deadline, **price** = целевая сумма).
3. Для равных долей: посчитать месячный план (цель / число месяцев, округлить при необходимости). Для разных: заполнить plan_amount по месяцам из анкеты или рассчитать по формуле.
4. Вставить в `dreams_steps` по одной записи на месяц: title (название месяца), plan_amount, fact_amount = 0, sort_order, deadline = выбранное число месяца (или последний день месяца).
5. Проверить отображение и логику Итога в кабинете.

## Инструкции для особых целей (индекс)

| rule_code / тип | Описание | Где инструкция |
|-----------------|----------|----------------|
| Финцель, равные доли | Сумма за период делится поровну по месяцам (например, 17 т.р./мес). Таблица ВЗНОС/СУММА/Итог, в расписании «Внеси N т.р.» | Выше: разделы 1–6. В `steps_rules`: category_code `finance`, rule_code `yearly_amount_by_month_equal`. |
| Финцель, разные доли | Сумма по месяцам задаётся вручную или по формуле (например, янв 0, фев 100 т.р., далее рост до суммы 3 млн). Тот же интерфейс, что и для равных. | Выше: разделы 1–6 и «Модалка при Добавить шаги» (п. 4a). В `steps_rules`: rule_code `yearly_amount_by_month_custom`. |
| Книги (books_reading) | Список книг, статусы (читаю/слушаю/завершил), виртуальные строки в расписании «Читать/Слушать … N мин», лог по дням. | Полная текстовая инструкция будет добавлена в этот раздел. В БД: `steps_rules.rule_code = 'books_reading'`, таблицы `dream_books`, `dream_books_log`. |

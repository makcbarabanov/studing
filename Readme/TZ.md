## 2. Мгновенное обновление при смене статуса бадди

**Задача:** Когда бадди меняет статус мечты, владелец должен видеть это **мгновенно** (без перезагрузки страницы).

**Реализация:** Polling, WebSocket или аналогичный механизм синхронизации.

---

## 3. Не сбрасывать позицию при действиях с мечтой

**Задача:** При любом действии (смена статуса, редактирование мечты по карандашу) пользователь **не должен** перекидываться на первую мечту.

**Сейчас:** Пользователь на 7-й мечте → меняет статус или нажимает карандаш → через ~7 сек его перекидывает на первую. Так быть не должно.

**Решение:** Сохранять текущую позицию/скролл/выбранную мечту при обновлении списка после API-запроса.

---

## 4. Разделить разовые и повторяющиеся шаги

**Задача:** Сейчас реализована схема для повторяющихся шагов. Нужна схема для **разовых** шагов: название + дата.

**Структура модалки «Добавить шаг»:**

1. **Название шага** — общее поле (всегда)
2. **Дата шага** — поле даты (всегда)
3. **Галочка «Повторять шаг»** — при включении показывается блок:
   - Дата начала (по умолчанию — текущая дата)
   - Дата окончания
   - Дни недели

**Визуал:** Блок «Повторять шаг» — серый (неактивный), когда чекбокс выключен. При включении чекбокса — блок становится активным.

**БД/API:** Уточнить, как хранить разовый vs повторяющийся (отдельные поля или флаг).

## 8. Промпт для фонового рисунка (Захваткин)

**Задача:** Написать промпт для генерации фонового изображения по совету Кости.

**Требования:** Вид сверху (космоснимок) тропического острова: маленький остров, пляж по периметру, пальмы по центру. Учесть **мобильную** и **браузерную** версию (размеры, crop).

**Промпт (EN, для Midjourney/DALL·E/Stable Diffusion):**

> Aerial satellite view of a small tropical island, top-down perspective. Sandy beach ring around the perimeter, turquoise shallow water, palm trees clustered in the center. Calm ocean, sunny day, soft clouds. Dreamy, inviting atmosphere. High resolution, photorealistic. Aspect ratio suitable for both desktop (wide) and mobile (portrait) — consider generating 16:9 and 9:16 variants or a square crop-friendly composition with the island centered.

**Промпт (RU, для русскоязычных генераторов):**

> Вид сверху, как на космоснимке: маленький тропический остров. Пляж по периметру, бирюзовая мелководная вода, пальмы в центре. Спокойный океан, солнечный день, лёгкие облака. Уютная, приглашающая атмосфера. Высокое разрешение, фотореалистично. Соотношение сторон под десктоп (широкий) и мобильный (вертикальный) — остров по центру, чтобы можно было обрезать под разные форматы.

## 9. Профиль и открытые мечты

### 9.1. Сообщение о заполнении профиля

**Задача:** При первом заходе в кабинет (и при неполном профиле) — сообщение: «Ваш профиль заполнен на N% — перейти к заполнению».

**Требования:** Аккуратно подобрать место (не перегружать интерфейс).

**Расчёт N%:** Какие поля учитывать? (имя, фамилия, город, telegram, vk, phone, свой вариант связи)

### 9.2. Проверка мессенджеров при открытой мечте

**Задача:** При переключении мечты в **открытую** (публичную) — проверять: заполнен ли хотя бы один способ связи (telegram, vk, phone, свой вариант). Если нет — **обязать** заполнить профиль (хотя бы один мессенджер) перед открытием.

**При закрытой мечте** — проверка не нужна (пользователь может не планировать открывать мечты).

**Где привязать:** Везде, где есть переключатель открытости мечты (модалки добавления/редактирования, клик по глазику в списке).

### 9.3. Текст блоков «Закрытая/Открытая мечта»

**Задача:** Сейчас текст слипается. Разнести по строчкам:

- **Закрытая мечта**  
  Видна только тебе и бадди.

- **Открытая мечта**  
  Попадает в витрину.

### 9.4. Убрать автоматический WhatsApp в витрине

**Проблема:** Сейчас при регистрации нового пользователя и добавлении новой мечты в витрине автоматически появляется кнопка «Написать в WhatsApp», хотя пользователь этого не указывал.

**Логика:** При регистрации пользователь указывает только телефон (для входа). При добавлении мечты и её открытии — проверка на наличие способа связи (чтобы помочь, нужен открытый контакт). WhatsApp не показывать автоматически из номера телефона. Пользователь должен явно выбрать, показывать ли WhatsApp (или добавить в «предпочтительный способ связи»).

**Решение:** Не показывать WhatsApp в витрине по умолчанию. Только если пользователь явно указал его как предпочтительный способ связи или выбрал в настройках.

### 9.5. Редактирование профиля: контакты

**Задачи:**
1. **Смена пароля** — перенести вниз (в форме редактирования профиля).
2. **Крестик для удаления** — у каждого мессенджера (Telegram, VK, телефон) кнопка ✕ для удаления. Нет WhatsApp — нажал ✕ и убрал.
3. **Добавить способ связи** — кнопка «Добавить способ связи». Открывает текстовое поле, где пользователь сам описывает, как с ним связаться (произвольный текст).
4. **При первой регистрации** — не подставлять WhatsApp автоматически. Номер телефона есть только для входа. В витрине контакты не показывать, пока пользователь не откроет мечту и не пройдёт проверку (9.2).

### 9.6. Предпочтительный способ связи

**Задача:** В личном кабинете — галочка или выпадающий список для выбора **предпочтительного способа связи** (при наличии более двух мессенджеров; телефон не считается).

**Зачем:** Чтобы в витрине «Хочу помочь» показывать один основной контакт, а не все подряд.

### 9.7. Новая мечта по умолчанию — закрытая

**Задача:** При создании новой мечты по умолчанию выбирать **«Закрытая мечта»**, а не открытая.

# Механика «Помогаю»: Готово / Вернуть / В процессе

Спецификация (ответы согласованы). См. также структуру интерфейса в **Readme/structure.md** (4 элемента: хедер, заголовки/меню, контент, футер).

---

## Согласованные ответы

1. **Несколько помощников** — да. Одной мечте могут помогать несколько человек. При нажатии владельцем «Принять» **все** помощники по этой мечте получают +1 к счётчику «помог» (у каждого в «Помог» добавляется эта мечта). Логика не зависит от количества помощников.

2. **Уведомления** — колокольчик в хедере. При появлении нового события (например, «Максим завершил твою мечту») у колокольчика появляется индикатор (зелёная точка, предпочтительно; или красная). По нажатию на колокольчик в **поле контента** (Элемент 3) отображается список уведомлений (не выпадающее меню, а смена контента).

3. **«Отказаться» (МВП)** — от всех помощников по этой мечте разом. Позже можно доработать до «от одного выбранного» и список «Помогают: Максим, Оля» с действиями по каждому.

4. **«Принять» в «В процессе»** — только владелец мечты. Бадди с правом редактирования (buddy_trust) пока не может нажимать «Принять».

5. **«В процессе»** — только фильтр/категория на витрине (мои мечты, которым помогают). Без смены `status_id` мечты до нажатия «Принять»; мечта остаётся «В работе», пока владелец не нажмёт «Принять» → тогда «Выполнено».

## Как я понял механизм

### Роли

- **Владелец** — автор мечты (`dreams.user_id`).
- **Помощник** — пользователь, нажавший «Хочу помочь» (есть запись в `user_dream_help_intent`).

### Витрина: фильтр «Помогаю»

- Показываются **чужие** мечты, по которым я помощник (я в `help_intent`, меня ещё нет в `helped`).
- В карточке **нет** кнопки «Хочу помочь».
- В карточке **две** кнопки:
  - **Готово!** — «я сделал, что мог, считаю помощь завершённой».
  - **Вернуть** — «больше не помогаю по этой мечте» (убрать меня из помощи: удалить из `help_intent` и при необходимости из `helped`).

### Действие помощника: «Вернуть»

- Мечта пропадает у меня из «Помогаю».
- У владельца мечта может пропасть из «В процессе», если я был единственным помощником.
- Статус мечты и записи в БД: только удаление связи «я помогаю» этой мечте.

### Действие помощника: «Готово!»

- Я (помощник) нажимаю «Готово!».
- У **владельца** появляется уведомление в духе: «Максим завершил твою мечту» с выбором:
  - **Принять** — мечта считается исполненной:
    - у владельца: мечта в «Помог» (или в счётчике «сбылось»),
    - у меня (помощника): мечта в «Помог»,
    - мечта переходит в статус «Выполнено», в `dreams_log` добавляется запись.
  - **Отправить на доработку** — «ещё не готово»:
    - мечта остаётся «В работе»,
    - я по-прежнему в «Помогаю», уведомление у владельца снимается (или помечается «отклонено»),
    - при желании я могу снова нажать «Готово!» позже.

### Витрина: новый фильтр «В процессе»

- Это **мои** мечты (я владелец), по которым есть хотя бы один помощник (есть записи в `help_intent` с `dream_id` моей мечты и другим `user_id`).
- В хлебных крошках витрины:  
  **Новые · Помогаю · Избранные · В процессе · Все** (и при необходимости «Помог»).
- В карточке мечты в «В процессе» у владельца две кнопки:
  - **Принять** — завершить мечту (как в сценарии «Готово! → Принять»: статус «Выполнено», `dreams_log`, помощник получает «Помог»).
  - **Отказаться** — отказаться от помощи **всех** помощников по этой мечте (МВП): удалить все записи `help_intent` по этому `dream_id`; у всех помощников мечта пропадает из «Помогаю».

### Связка «Готово!» и «В процессе»

- Когда помощник нажал «Готово!», владелец видит это в «В процессе» (или в уведомлении — см. вопрос 2): та же мечта с подписью «Максим завершил твою мечту» и кнопками **Принять** / **Отправить на доработку**.
- То есть «Принять» из «В процессе» и «Принять» из уведомления после «Готово!» делают одно и то же: мечта завершается, помощник получает «Помог».

## Что нужно в БД/API (кратко)

- **Состояние «помощник нажал Готово!»**  
  Чтобы владелец мог показать «Принять / На доработку», нужно хранить факт запроса на завершение. Варианты:
  - отдельная таблица, например `user_dream_completion_request (helper_user_id, dream_id, requested_at)`, или
  - колонка в `user_dream_help_intent`, например `completion_requested_at` (NOT NULL = ждёт решения владельца).
- **Фильтр «В процессе»**  
  Список и счётчик: мечты, где `dreams.user_id = я` и существует запись в `user_dream_help_intent` с этим `dream_id` (и при необходимости не в `helped`). В ответ витрины для владельца добавлять список «моих» dream_id с помощниками (и флаг «есть запрос на завершение»).
- **API**  
  - Помощник: `POST /dreams/{id}/completion-request` («Готово!»), при «На доработку» — `DELETE .../completion-request` (или PATCH с флагом).
  - Владелец: «Принять» = завершить мечту (обновить статус, `dreams_log` по одному разу на каждого помощника или одна запись — уточнить; **все** помощники переносятся в `helped`); «Отказаться» = удалить **все** записи `help_intent` по этому `dream_id` (МВП).

## Варианты улучшений и краевые случаи

1. **Таймаут запроса «Готово!»**  
   Если владелец долго не заходит, запрос «Максим завершил» может висеть неделями. Имеет смысл показывать дату запроса («Максим отметил готовность 15.02») и при желании потом добавить автосброс или напоминание.

2. **Повторное «Готово!» после «На доработку»**  
   После «Отправить на доработку» помощник остаётся в «Помогаю» и может снова нажать «Готово!». Один активный запрос на завершение на пару (dream_id, helper_id) — при новом «Готово!» обновлять `requested_at`.

3. **Завершение мечты владельцем без «Готово!»**  
   В «В процессе» владелец может нажать «Принять» и без запроса от помощника (например, сам довёл мечту до конца при поддержке). Мечта завершается; **все** текущие помощники по этой мечте получают «Помог» (добавить в `helped`; в `dreams_log` — одна запись с `fulfilled_by_user_id` = владелец или по одной записи на каждого помощника — уточнить при реализации).

4. **«Отказаться» и «На доработку»**  
   «Отказаться» = «я не хочу твоей помощи» (убираем из help_intent). «На доработку» = «помощь ещё не закончена» (оставляем в help_intent, сбрасываем запрос на завершение). Разные действия, разный тон.

5. **Счётчики в хедере**  
   Сейчас есть счётчики типа «N выполнил» / «N мечт сбылось». Имеет смысл явно решить: показывать ли отдельно «У вас 1 мечта в процессе (ждёт вашего решения)» или только внутри витрины в «В процессе».

6. **Мобильная витрина**  
   Те же фильтры и те же кнопки (Готово! / Вернуть в «Помогаю»; Принять / Отказаться в «В процессе»), чтобы сценарии не расходились с десктопом.

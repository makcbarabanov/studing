# Roadmap — Завтрак исполняющий желания

Соцсеть для целеустремлённых людей. Сюда заносим шаги и идеи по развитию.  
Шаги внутри разделов помечаем как **плановые** или **выполненные** — изменения только с твоего разрешения (ты говоришь «пометь шаг выполненным» или соглашаешься, когда спрашиваю).

## Расписание (ранее «Шаги»)

**Цель:** раздел «Расписание» = сводка всех шагов по всем мечтам (одни и те же данные, что в «теле» мечты). Дорожная карта: открыл, посмотрел что сделать сегодня, отметил и пошёл дальше.

- Пункт меню **«Расписание»** (переименовать с «Шаги»). Все шаги из всех мечт отображаются здесь в едином списке.
- Шаги в Мечтах (аккордеон) и шаги в Расписании — **одни и те же записи** (`dreams_steps`). Подробно: [Readme/structure.md](structure.md) → «Шаги в Мечтах и Расписание».
- **Для шагов финцели в Расписании:** одна строка на месяц — **«Внеси N т.р.»** (понятная формулировка; в Мечтах та же цель — в виде таблицы). Шаг виден в списке **в день дедлайма** (31 марта для марта). Дедлайн = последний день месяца. В начале списка — актуальные/сегодняшние шаги; фильтр **«Просроченные»**. Модалка при галочке только для фин-шагов: «Вносишь ровно N руб?» → «Да» или другая сумма; сохранение `fact_amount`, следующий месяц с новой суммой. Ввод факта в таблице мечты автоматически закрывает шаг в Расписании.

| Шаг | Статус |
|-----|--------|
| Переименовать раздел «Шаги» в «Расписание» (меню, заголовки) | Плановый |
| Добавить дедлайн у шага (БД + API + UI в аккордеоне) | Плановый |
| Редактирование шага на лету (название, дедлайн) | Плановый |
| Раздел Расписание: список всех шагов (Шаг + дедлайн), как в аккордеоне | Плановый |
| В Расписании: галочка выполнен, крестик удалить (как в аккордеоне) | Плановый |
| Фильтр: все / текущие / выполненные (по умолчанию — текущие) | Плановый |
| Выполненные шаги — светлее, курсив; удалённые — отдельно, восстановить | Плановый |
| Финцель в Расписании: одна строка «Внеси N т.р.» на месяц, видна в день дедлайма (последний день месяца), модалка при галочке, синхрон с таблицей в Мечте | Плановый |
| Сортировка Расписания: сначала актуальные/сегодняшние шаги; фильтр «Просроченные» | Плановый |

## Бадди

**Цель:** довести раздел до полного цикла: аватар и статистика бадди в хедере, назначение и подтверждение бадди. Поддержка **нескольких бадди** (можно отправить несколько запросов и иметь несколько принятых бадди; чем больше — тем лучше).

- Шаги уже сделанные: связь buddy_id, просмотр мечт бадди, buddy_trust, редактирование мечт бадди.
- **Схема:** сейчас в `users` одно поле `buddy_id` (один бадди). Для нескольких бадди нужно изменить модель: отдельная таблица связей (например `user_buddies`: user_id, buddy_id, trust, статус запроса) или массив/JSONB. В roadmap зафиксировано: переход к «несколько бадди».
- Идеи: аватарка бадди(ев) и их статистика в хедере; функция **назначения** (отправить запрос одному или нескольким) и **подтверждения** (принять/отклонить; можно принять хоть 100 бадди).

| Шаг | Статус |
|-----|--------|
| buddy_id, просмотр мечт бадди | Выполнено |
| buddy_trust, редактирование мечт бадди | Выполнено |
| Схема БД: несколько бадди (таблица связей user_buddies или аналог) | Плановый |
| Аватарка бадди(ев) и их статистика в хедере | Плановый |
| Назначение бадди: запрос «стать бадди», можно отправить нескольким | Плановый |
| Подтверждение бадди: принятие/отклонение; можно иметь много бадди | Плановый |

## Аналитика

**Цель:** продумать, как считать и показывать «мечта сбылась N раз», статусы «сбылось и завершилось» / «сбылось и продолжает», «скольким людям я помог».

- Пример: мечта «чтобы родители при жизни слышали „спасибо за сына“» — сбылась 3 раза (Светлана Щербинина, Айгуль, ещё человек вне системы). Нужна **цифра 3** и возможность не «закрывать» мечту — она продолжается.
- Статусы: **сбылось и завершилось** / **сбылось и продолжает** — придумать названия, иконки, куда вписать число сбываний (например, 3).
- Учёт: **скольким людям я помог** (исполнил чужие мечты), **где это отображать**.
- Задача — кропотливая проработка, чтобы в итоге получить простой интерфейс.

| Шаг | Статус |
|-----|--------|
| Определить статусы: «сбылось и завершилось» / «сбылось и продолжает» (названия, иконки) | Плановый |
| Хранить и показывать «мечта сбылась N раз» (в т.ч. для мечт «продолжает») | Плановый |
| Учёт «скольким людям я помог» (исполнение чужих мечт) | Плановый |
| Место отображения: цифры сбываний, помощь другим (хедер, профиль, отдельный блок) | Плановый |

## Витрина мечт

Сюда собираются **все открытые мечты** (is_public): общая лента/витрина, чтобы целеустремлённые видели мечты друг друга и вдохновлялись.  
Витрина: **dreams.html** — публичная (без авторизации): просмотр, «Хочу помочь» → регистрация в ЛК. В ЛК — витрина по клику «N выполнил» (с фильтрами, избранным, просмотрами).

| Шаг | Статус |
|-----|--------|
| Витрина: страница / и /dreams.html, данные из БД (GET /dreams/showcase), фильтры, карточки, модалка «Как помочь», ссылка на личный кабинет | Выполнено |
| Витрина внутри кабинета: клик «N выполнил» → витрина; клик «N мечт сбылось» → возврат; фильтры Новые/Помогаю/Все/Избранные/Просмотренные; просмотры, избранное, help_intent в БД; модалка с Telegram/WhatsApp/VK (users.telegram, users.vk) | Выполнено |

## Мобильная версия

Текущая вёрстка не помещается на мобильном; этому разделу уделим отдельное внимание.

- **Порядок:** сначала **браузер со свайпами** (адаптивная вёрстка, один экран — одна мечта, свайп влево/вправо), потом при необходимости — отдельное мобильное приложение.
- Идея: на одном экране — **одна мечта**: описание, статус, дедлайн, кнопки редактирования. Другая мечта — **свайп влево/вправо** (реализуем в браузерной мобильной версии).

| Шаг | Статус |
|-----|--------|
| Адаптивная вёрстка под мобильный экран (index.html, dreams.html: брейкпоинты 768px, 480px; хедер, меню, таблица, витрина, модалки, touch-зоны 44px) | Выполнено |
| Один экран — одна мечта, свайп для переключения (в браузере) | Плановый |
| Позже: мобильное приложение (при необходимости) | Идея |

**Цель:** научиться автоматически раскладывать цели (особенно финансовые) на понятные шаги.

| Шаг | Статус |
|-----|--------|
| Описать и реализовать базовую систему разбиения целей на шаги (по типам целей: финансовые, спортивные и т.д.) | Плановый |

### Анкета для финансовых целей

Спецификация модалки при «Добавить шаги» для финцели: [Readme/rules.md](rules.md) — раздел «Модалка при Добавить шаги (анкета финцели)». Вопросы: заработать/накопить, нужная сумма, срок (дата + подсказки «12 месяцев» / «до конца года»), доли равные/разные (при разных — поля по месяцам или формула: 1-й месяц 0, 2-й A, множитель N, сумма = цель), число внесения (день месяца в расписании).

| Шаг | Статус |
|-----|--------|
| Спроектировать анкету при создании финансовой цели (сумма, валюта, период, старт/конец, предпочтения по распределению) | Плановый |
| На основе ответов анкеты автоматически генерировать шаги (например, 12 месяцев с планом/фактом) | Плановый |
| Модалка «Добавить шаги» для финцели: все пункты анкеты (равные/разные, срок с подсказками, число внесения) | Плановый |
| Правило `yearly_amount_by_month_custom` в steps_rules + сохранение в dreams.settings; при «разные» — ввод по месяцам или формула (янв 0, фев A, ×N, сумма = цель) | Плановый |

### Идентификация мечты как «особой» (rule_code)

**Цель:** определить, как новая мечта получает привязку к правилу (`dreams.rule_code`), чтобы при «Добавить шаг» система подставляла нужную инструкцию и интерфейс (финансы, книги и т.д.).

- Варианты: ручное назначение при создании/редактировании мечты (выбор типа декомпозиции); привязка по категории (например, категория «Книги» → rule_code `books_reading`); будущая анкета/мастер при создании мечты.
- Механизм пока в разработке (с Атласом); идеи и уточнения — в этом пункте roadmap.

| Шаг | Статус |
|-----|--------|
| Определить способ присвоения rule_code новой мечте (ручной выбор, по категории, анкета) и момент срабатывания при «Добавить шаг» | Плановый |
